# 智信云 ERP 项目架构分析

## 一、顶层目录架构

```
zhixincloud/
├── public/                  # 静态资源（不经过 webpack 处理）
│   ├── 404.html             # 404 页面
│   ├── favicon.ico          # 网站图标
│   ├── ant_icon/            # Ant Design 图标字体
│   ├── zx_icon/             # 智信云自定义图标字体（SVG + JS）
│   ├── tinymce/             # 富文本编辑器（TinyMCE）及其插件/皮肤
│   └── mock/                # Mock 数据（flow.json, system.json）
│
├── src/                     # 核心源码
│   ├── main.ts              # 应用入口：初始化 Vue、Router、Store、插件
│   ├── App.vue              # 根组件：<router-view> + 全局抽屉 + 通知栏
│   ├── premission.ts        # 路由守卫：白名单鉴权逻辑
│   │
│   ├── api/                 # 接口层：所有后端 API 请求封装
│   ├── assets/              # 静态资源（经 webpack 处理：图片、SVG）
│   ├── components/          # 全局可复用组件
│   ├── views/               # 页面视图（按业务模块分目录）
│   ├── router/              # 路由配置（Vue Router）
│   ├── store/               # 状态管理（Vuex）
│   ├── utils/               # 工具函数库
│   ├── utils1/              # 另一套工具函数（含独立 request 封装）
│   ├── helpers/             # 辅助函数（DOM、拖拽、响应式、通用工具）
│   ├── services/            # 服务层（portal 模块的 Service 封装）
│   ├── plugins/             # 第三方插件（vnet-ui 组件库）
│   ├── style/               # 全局样式（Less）
│   └── guide/               # 新手指引（Intro.js 步骤配置）
│
├── node_modules/            # 依赖包
├── package.json             # 项目配置与依赖声明
├── vue.config.js            # Vue CLI 构建配置
├── tsconfig.json            # TypeScript 配置
├── babel.config.js          # Babel 编译配置
└── .eslintrc.js             # ESLint 代码规范
```

### 各目录职责说明

| 目录 | 职责 |
|---|---|
| **api/** | 封装所有后端 HTTP 请求，按业务域拆分：`login.ts`（登录/用户）、`menu.ts`（菜单/权限）、`feedback.ts`（反馈）、`notice.ts`（公告）、`template.ts`（模板）、`view.ts`（视图）、`applicationCenter.ts`（应用中心）等 |
| **assets/** | 存放经 webpack 处理的静态资源，主要是图片和 SVG 图标，按功能分子目录（`applyCenter/`、`icons/`、`img/`） |
| **components/** | 全局可复用组件：`SvgIcon`（SVG 图标）、`echart`（图表封装）、`skeleton`（骨架屏）、`vueGuidLayout`（网格拖拽布局）、`CheckPersonByDep`（人员选择器）、`TurnFlow`（流程转交）、`NewsPublic`（新闻发布相关）、`fullScreen`（全屏）等 |
| **views/** | 页面视图组件，按业务模块分目录：`layout/`（壳子布局）、`workbench/`（工作台）、`feedback/`（反馈）、`help/`（帮助文档）、`feishu/`（飞书集成）、`role/`（角色管理）、`product/`（产品/视图）、`newsNotice/`（新闻公告）、`superQuery/`（超级查询）等 |
| **router/** | Vue Router 路由配置，定义所有页面路径、名称、懒加载方式及 Layout 嵌套关系 |
| **store/** | Vuex 状态管理，分三个模块：`user`（用户信息/登录态）、`tagView`（多页签管理）、`setting`（用户偏好设置：主题、侧边栏、页签模式等） |
| **utils/** | 工具函数：`request.ts`（axios 封装）、`util.ts`（通用工具）、`rem.js`（移动端适配）、`feishu.js`（飞书集成）、`dom.js`（DOM 操作）、`index.js`（URL 解析等） |
| **helpers/** | 辅助工具类：`utils.ts`（大型通用工具集）、`DOM.ts`（DOM 操作）、`draggableUtils.ts`（拖拽）、`responsiveUtils.ts`（响应式） |
| **services/** | 服务层封装，将 API 调用组织为 Service 类（`portal/` 下有 `User.ts`、`NewsPublic.ts`、`TestService.ts` 等） |
| **style/** | 全局 Less 样式入口文件 `index.less` |
| **plugins/** | 第三方 UI 库 `vnet-ui` 的组件注册文件 |
| **guide/** | 新手指引的 Intro.js 步骤定义 |

## 三、"壳子"渲染流程

### 第 1 步：`main.ts` 启动 — 装配"骨架"

```
main.ts 加载顺序：
1. 引入 babel-polyfill（兼容 IE）
2. 创建 Vue 实例
3. 注册全局插件：VNETUI（自研组件库）、VNETUIBusiness（业务组件库）、skeleton（骨架屏）
4. 注册全局组件：SvgIcon（SVG 图标）
5. 挂载全局原型：$echarts（图表）、$HandleMessageBox（确认弹框）
6. 注入 portalServices 到 provide（依赖注入，子组件可直接获取）
7. 引入 rem.js（移动端适配）
8. 引入全局样式 index.less
9. 引入 premission.ts（注册路由守卫）
10. 监听 window.message 事件（跨窗口通信，处理退出登录）
11. 渲染根组件 App → mount 到 #app
```

### 第 2 步：`premission.ts` 路由守卫 — 鉴权

```typescript
// 白名单：这些路径不需要鉴权，直接放行
whiteList = ["/login", "/loginNew", "/autoLogin", "/authorization"]

router.beforeEach((to, from, next) => {
  if (whiteList.includes(to.path)) {
    next()                    // 白名单路径 → 直接放行
  } else {
    next({ query: to.query }) // 非白名单 → 携带 query 参数重新触发路由
  }
})
```

**关键点**：这个项目的鉴权逻辑比较特殊——它没有在路由守卫里主动跳转登录页。非白名单路径会 `next({ query: to.query })` 重新进入当前路由，此时 `App.vue` 的 `created` 中会通过 `window.onlineUser` 检查用户状态。真正的登录态校验发生在 **API 层**——当接口返回 401/403 时，`App.vue` 监听的 `AJAXERROR` 事件会触发 `store.dispatch('quit')` 跳转到 `#/login`。

### 第 3 步：`App.vue` 根组件渲染 — 全局容器

```
App.vue 模板结构：
<div id="app">
  <router-view />                    ← 核心：路由出口，渲染匹配的页面组件
  <global-notify />                  ← IE 浏览器顶部通知栏
  <el-drawer v-for="drawers" />      ← 动态抽屉面板（用于打开第三方页面）
</div>
```

`App.vue` 在 `created` 生命周期中做了这些事：
- 将 `window.onlineUser` 暴露给全局（供子应用/iframe 读取）
- 注册 `window.openTags()` 全局方法（打开新页签/抽屉）
- 注册 `window.closeTagView()` 全局方法（关闭页签）
- 监听 `AJAXERROR` 总线事件（401/403 → 退出登录）
- 飞书环境下启动定时续 session（每小时一次）
- 注册全局 loading 控制方法

### 第 4 步：`Layout` 布局组件渲染 — 壳子

当用户访问 `/` 时，路由匹配到 `Layout` 组件（`views/layout/layout.vue`）：

```
Layout 模板结构：
<div class="bannerLayout">
  <global-banner />                  ← 顶部 Banner（可关闭）
  <global-header />                  ← 顶部导航栏（Logo、全局菜单、搜索、通讯录、设置）

  <div class="app-content">
    <div class="sideBar">            ← 左侧一级菜单栏（流程中心、客户服务中心、办公中心...）
      <menubar />                    ← 二级菜单弹出层（悬浮/点击展开）
    </div>

    <div class="rightCon">
      <router-view />                ← 核心：二级路由出口，渲染具体业务页面
    </div>
  </div>

  <updateDialog />                   ← 版本更新提示弹框
</div>
```

`Layout` 在 `mounted` 中的初始化流程：
1. `getSysSetting()` → 从服务器拉取用户偏好设置（主题、侧边栏状态、页签模式等）
2. `GetUserInfo()` → 获取当前用户信息存入 Vuex
3. `GetDataPerson()` → 获取数据中心人员权限
4. `selectPersonRole()` → 查询用户角色
5. `initMenu()` → 初始化菜单数据
6. `getParentSysMenu()` → 获取一级菜单列表，渲染左侧侧边栏（流程中心、客户服务中心、办公中心、网络运营中心、BI 等）
7. 检查是否需要显示版本更新弹框 / 新手指引

### 第 5 步：加载具体业务页面

`Layout` 内部的 `<router-view>` 根据当前路由渲染对应的子页面组件（如 `/dashboard` → `workbench/index.vue`）。这些页面组件都是**懒加载**的（`() => import(...)`），webpack 会将其拆分为独立的 chunk，按需加载。

### 完整流程图

```
浏览器访问 → index.html → 加载 main.ts
                              │
                              ▼
                    创建 Vue 实例（Router + Store + 插件）
                              │
                              ▼
                    引入 premission.ts → 注册 beforeEach 守卫
                              │
                              ▼
                    渲染 App.vue → <router-view>
                              │
                              ▼
                    路由守卫鉴权 ──→ 白名单? → 放行
                         │                      │
                         └── 非白名单 → next({query}) 重入
                                              │
                                              ▼
                                    Layout.vue 渲染（壳子）
                                              │
                                    ┌─────────┼─────────┐
                                    ▼         ▼         ▼
                              globalBanner  globalHeader  sideBar
                              （顶部Banner） （顶部导航）  （左侧菜单）
                                              │
                                              ▼
                                    加载用户信息 + 菜单数据
                                    （Vuex user/setting 模块）
                                              │
                                              ▼
                                    <router-view> 渲染子页面
                                              │
                                              ▼
                                    懒加载具体业务组件
                                    （workbench / feedback / help ...）
```

**一句话总结**：`main.ts` 装配骨架 → `premission.ts` 路由守卫做白名单鉴权 → `App.vue` 提供全局容器和跨窗口通信 → `Layout.vue` 渲染顶部导航+左侧菜单的"壳子"并拉取用户/菜单数据 → 内部 `<router-view>` 懒加载具体业务页面。
